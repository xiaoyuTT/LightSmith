### 📹 第二步："摄像头"如何工作 (数据采集与上传)

一旦"摄像头"（即 `@traceable` 装饰器或LangChain的自动钩子）被部署，它就会忠实地记录下每一次执行的详细信息：

- **记录一个"运行" (Run)**：Agent执行的每一个步骤，在LangSmith中都被称为一个 **"运行"**。例如：
  - **一次LLM调用**：会记录输入的`prompt`、返回的`output`、消耗的`token`数量、模型名称、调用的耗时等。
  - **一次工具调用**：会记录调用的`tool`名称、传入的`参数`、工具返回的`结果`或`错误`信息。
  - **一次链的调用**：会记录整个子流程的开始和结束。
- **结构化上传**：所有这些"运行"数据，包括它们的输入、输出、开始/结束时间、元数据（如会话ID、用户标签等），都会被实时、异步地打包发送到LangSmith的后台服务。这个过程是异步的，所以**基本不会影响你主应用的性能**。

### 🎯 方式一：自动拦截（对LangChain用户最友好）

如果你用的是LangChain或LangGraph框架，几乎不需要改代码。这是最“自动”的方式。

**核心原理**：只要设置了环境变量 `LANGSMITH_TRACING=true`，LangChain在执行任何组件（如`LLMChain`、`ChatOpenAI`、`Tool`）时，都会检测到这个标志。然后，它会自动调用内置的`LangChainTracer`回调处理器，把所有关键信息（输入、输出、耗时等）打包发往LangSmith服务器。

**具体是怎么拦截的？**

- **LLM调用**：当你执行 `llm.invoke("Hello")` 时，LangChain内部的`BaseChatModel`会触发一个回调事件，`LangChainTracer`捕获到这个事件，记录下`prompt`、`response`、`model name`、`tokens`等信息。
- **工具调用**：当Agent决定调用`search`工具时，`Tool`类的执行方法同样会触发回调，捕获`tool name`、`input arguments`和`output`。
- **链/Agent调用**：整个`AgentExecutor`或`LLMChain`的调用被视为一个“父运行”（Parent Run），它会包含上述所有子步骤（Child Runs），形成你看到的“追踪树”。

**这种方式的好处是**：零侵入，只要设置好环境变量，所有用LangChain写的代码都会被自动追踪。

### 🛠️ 方式二：手动包装（对非LangChain用户更通用）

如果你不用LangChain，或者想追踪自己写的函数，LangSmith提供了更灵活的SDK，核心是两种工具：

1. **`@traceable` 装饰器（Python）**
   这是最推荐的方式。你只需要在你关心的任何函数上面加一行 `@traceable`，这个函数的调用、输入、输出就会被自动记录下来。

   python

   ```
   from langsmith import traceable
   import openai
   
   @traceable  # 👈 就是这个“摄像头”
   def call_llm(messages):
       # 这里可以是你自己的OpenAI调用逻辑，甚至是复杂的业务代码
       client = openai.OpenAI()
       return client.chat.completions.create(model="gpt-4", messages=messages)
   
   # 调用函数时，LangSmith会自动捕获它的执行信息
   call_llm([{"role": "user", "content": "Hello!"}])
   ```

   

   `@traceable`的原理类似于Python装饰器，它在函数执行**前**开始一个“运行”（Run），记录开始时间和输入；在函数执行**后**结束这个“运行”，记录输出、耗时，并自动捕获内部抛出的异常。如果这个函数内部又调用了另一个被`@traceable`装饰的函数，它会自动形成父子关系的追踪树。

2. **包装器（Wrapper）**
   对于OpenAI、Anthropic等主流SDK，LangSmith提供了现成的包装器。

   python

   ```
   from langsmith.wrappers import wrap_openai
   import openai
   
   # 用wrap_openai包装一下原来的客户端
   client = wrap_openai(openai.OpenAI())
   
   # 之后的所有调用都会被自动追踪
   client.chat.completions.create(...)
   ```

   

   `wrap_openai`的原理是动态替换或增强`openai.OpenAI`类的方法，在原始的`chat.completions.create`调用前后插入追踪代码。

### 📝 幕后功臣：数据结构化

无论通过哪种方式捕获，LangSmith最后一步都需要把这些杂乱的数据“标准化”，才能在UI上渲染成漂亮的追踪树。

- **输入/输出格式**：LangSmith定义了一套自己的消息格式。如果你用的是LangChain或`wrap_openai`，SDK会自动帮你转换。如果你用`@traceable`追踪自定义函数，可以通过`process_inputs`和`process_outputs`参数来告诉LangSmith如何解析你的数据。
- **元数据标识**：为了让UI知道这是个LLM调用而不是普通函数，LangSmith会要求或自动补充元数据，比如`ls_provider`（模型供应商）和`ls_model_name`（模型名称）。

### 💎 总结：一张表看懂区别

| 特性           | **自动拦截 (LangChain)**                                     | **手动包装 (通用SDK)**                                       |
| :------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **实现方式**   | 通过环境变量`LANGSMITH_TRACING=true`触发LangChain内置的`LangChainTracer`回调 | 显式使用`@traceable`装饰器或`wrap_openai`包装器              |
| **代码侵入性** | **几乎为零**，无需修改现有代码逻辑                           | **低**，只需在函数定义处添加装饰器或初始化时包装客户端       |
| **适用场景**   | 使用LangChain或LangGraph构建的应用                           | 任何Python/JS应用，包括原生OpenAI调用、自定义逻辑、RAG流程等 |
| **核心原理**   | 框架层面的回调机制（Hook）                                   | 装饰器增强（AOP）和SDK方法增强（Wrapper）                    |

### 🚀 推荐的工作流程

1. **开发阶段**：直接在代码开头设置好环境变量，这样所有LangChain的调用都能自动生成追踪，对于你自己的业务函数，顺手加上`@traceable`装饰器。
2. **生产环境**：可以通过`with tracing_context(enabled=True)`这样的上下文管理器，有选择性地只追踪特定会话或特定比例的用户请求，在成本和调试需求之间取得平衡。

LangSmith通过这些机制，把Agent执行过程中那些原本“隐形”的步骤，变成了可审计、可调试的资产。希望这个解释能帮你彻底理解它的工作原理。

## 🚀 示例场景：一个旅行规划Agent

假设你构建了一个Agent，它需要：

1. 理解用户提问（第一轮LLM调用）
2. 调用天气API查询目的地天气（工具调用）
3. 调用航班API查询机票（工具调用）
4. 综合信息生成最终建议（第二轮LLM调用）

你在代码中用`@traceable`装饰器追踪了整个处理函数：

python

```
from langsmith import traceable
import openai
from langsmith.wrappers import wrap_openai

# 包装OpenAI客户端以自动追踪
client = wrap_openai(openai.OpenAI())

@traceable(name="旅行规划Agent", run_type="chain")
def travel_agent(destination, date):
    # 第一轮：理解用户意图
    intent_response = client.chat.completions.create(
        model="gpt-4",
        messages=[{"role": "user", "content": f"我想去{destination}旅行，日期是{date}"}]
    )
    
    # 调用天气工具（模拟）
    weather = get_weather(destination, date)  # 这个函数也被@traceable装饰
    
    # 调用航班工具（模拟）
    flights = search_flights(destination, date)  # 这个函数也被@traceable装饰
    
    # 第二轮：生成最终建议
    final_response = client.chat.completions.create(
        model="gpt-4",
        messages=[
            {"role": "user", "content": f"目的地天气：{weather}，航班信息：{flights}。请给出旅行建议"}
        ]
    )
    
    return final_response.choices[0].message.content
```

### 3️⃣ 输入/输出列的自定义预览

### 4️⃣ 筛选与分组：元数据和标签

你可以在UI左侧的筛选面板，按各种条件过滤Trace：

- **按错误**：勾选 `Error` 只看失败的Trace
- **按元数据**：`metadata.app_version = "v2.0"` 对比版本表现

如果你在代码中添加了metadata：
